Role Permissions

Note : This document may refer to Carbon components which are not used in your currently running Carbon instance. So please consider permissions only on Carbon components that you currently have.

  1. Login to admin console

    2. A Role must have this permission to log into the admin console. Without logging into the console, you can only have following facilities. If your Carbon instance doesn't contain any of the following functionalities, you won't have permissions to perform anything when you are not logged in.

    • View a list of services.
    • Access WSDL11 and WSDL12 of those services.
    • Use following tools.
      - WSDL2Java, Java2WSDL, WSDL Converter, Try It, Service Validator, Module Validator

  2. Manage system configuration

    3. A Role must have this permission to use following functionalities.

    • Throttling
    • Caching
    • Reliable Messaging
    • Operation Management
    • Governance
    • Scheduled Tasks
    • Module Management
    • Transport Management
    • JavaScript Services
    • Eventing
    • Logging
    • XKMS configuring
    • Data Service
    • Bam
    • Configure Synapse
    • Identity
    • Registry
      • Access to Resource Browser
      • Manage properties
      • Manage Activities
      • Manage Association
      • Resource Search
      • Registry Info
      • Registry Profiles
    • Governance
      • WSDL import
      • Gadget Source
      • Life cycles
      • Service Addition

  3. Manage security

    4. A Role must have this permission to use following functionalities.

    • Security
    • Manage Profiles
    • STS
    • Claim Management

  4. Upload services

    5. A Role must have this permission to upload following types of services.

    • JAR Services
    • AAR Services
    • POJO Services
    • JAXWS Services
    • EJB Services
    • Axis1 Services
    • BPEL Processes

  5. Manage services

    6. A Role must have this permission to use following functionalities.

    • All kinds of Axis services (JAR, AAR, POJO, JAXWS, EJB, Axis1)
    • Proxy Services
    • Modules
    • Cluster Management

  6. Manage mediation

    7. A Role must have this permission to use all kinds of mediation functionalities.

    • Sequences
    • Mediation
    • Local Entry

  7. Monitor system

    8. A Role must have this permission to use monitoring capabilities.

    • System Statistics
    • Logging
    • Mediation Statistics
    • Mediation Tracer
